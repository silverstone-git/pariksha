# Pariksha - Pending Tasks & Known Limitations

## 1. Advanced Exam Engine (Sections, MSQs, NATs)
Currently, Pariksha applies a uniform global marking scheme and only supports standard MCQs. To accurately simulate exams like GATE, CSIR NET, and TIFR, the engine needs an architectural rewrite to support:

- **Varying Marking Schemes**: Different positive/negative marks per section (e.g., TIFR Section A is +3/-1, Section B is +5/0).
- **Question Types**:
  - MSQs (Multiple Select Questions): Multiple correct options, no negative marking (used in GATE).
  - NATs (Numerical Answer Type): Text input for decimal/integer answers, no negative marking (used in GATE).
- **Attempt Limits**: Exams like CSIR NET require answering only a subset of questions (e.g., max 15 out of 20 in Part A). The engine must enforce and cap the graded questions per section.

## 2. Centralized Question Bank API (Production Readiness)
The "Auto-Generate Physics Exam" feature (Presets) currently fetches questions from the local development server (`cli/question_bank/` via Vite proxy). This is nonsensical for a deployed production app, as end-users will not have these local JSON files.

- **Problem**: The client app cannot generate preset exams in production because the question bank is local.
- **Proposed Solution**: 
  Develop a dedicated API on the main server (e.g., `https://outsie.aryan.cfd/api/question_bank`) to act as the unified source of truth.

  **Required API Endpoints & Specifications:**

  1. **Fetch Topic List & Stats (Used by Admin Panel & Presets)**
     - `GET /api/question_bank/topics`
     - **Output**: 
       ```json
       [
         { "name": "Quantum Mechanics", "slug": "quantum_mechanics", "count": 40 },
         { "name": "Classical Mechanics", "slug": "classical_mechanics", "count": 25 }
       ]
       ```

  2. **Sample Questions from a Topic (Used by Frontend `generatePresetExam`)**
     - `GET /api/question_bank/sample?topic=quantum_mechanics&count=10`
     - *Note: Server-side sampling is preferred to prevent the client from downloading thousands of questions just to pick a few.*
     - **Output**: Array of `Question` objects.
       ```json
       [
         {
           "question": "What is the commutator $[x, p]$?",
           "options": [{ "label": 1, "value": "$i\\hbar$" }, { "label": 2, "value": "$0$" }],
           "answer_label": 1,
           "topic": "Quantum Mechanics",
           "explanation": "Fundamental commutation relation...",
           "image_path": null
         }
       ]
       ```

  3. **Append New Questions (Used by Local Pariksha Admin Panel / CLI)**
     - `POST /api/question_bank/topics/{slug}`
     - **Headers**: `Authorization: Bearer <ADMIN_SECRET>` (Crucial: prevents public pollution of the bank).
     - **Input**: Array of new `Question` objects generated by the AI CLI.
     - **Action**: The server appends these new questions to the database/JSON file for that specific topic.
     - **Output**: `{ "success": true, "added": 20, "total": 60 }`

  **Workflow Shift**: 
  - The local `generate_question_bank.py` script will generate questions, save a local backup, and then immediately `POST` to the remote server using the admin secret.
  - The deployed frontend will hit the `GET /sample` endpoint for each topic required by a preset (e.g., GATE needs 10 Quantum questions, it makes one API call for them) instead of fetching local static JSON files.
