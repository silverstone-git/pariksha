## 3. Scientific Visualization & R2 Hosting (Infrastructure)
The multimodal generator uses Cloudflare R2 to host scientific diagrams.
- **Current State**: Images are uploaded to a dedicated `pariksha` bucket with public dev URLs.
- **Key Insight (The 401 Issue)**: Cloudflare R2 does **not** support S3-style Object ACLs (`public-read`). Public access must be enabled at the **Bucket Level** via the Cloudflare Dashboard.
- **Potential Solutions**:
  1. **Dedicated Public Bucket**: (Current Approach) Create a bucket specifically for public assets. Keep sensitive data in a separate private bucket.
  2. **Worker Gatekeeper**: Use a Cloudflare Worker to serve only specific prefixes (e.g., `/pariksha/*`) while keeping the rest of the bucket private.
- **Pitfalls & Security Risks**:
  * **Bucket Exposure**: Enabling public access on a bucket containing private data (like `outstagram`) exposes everything to guessable URLs.
  * **Token Scoping**: Migration scripts will fail with `AccessDenied` if the R2 API Token is restricted to a single bucket. Tokens must be scoped to "All Buckets" or specifically include the new destination.
  * **CORS**: Ensure the R2 bucket has a CORS policy that allows the frontend domain.
